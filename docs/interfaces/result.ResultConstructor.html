      Note: While you may create the Result type via normal JavaScript class construction, it is not recommended for the functional style for which the library is intended. Instead, use ok.

      // Avoid:
      const aString = new Result.Ok('characters');

      // Prefer:

      Note that you may explicitly pass {@linkcode Unit.Unit Unit} to the Ok constructor to create a Result<Unit, E>. However, you may not* call the Ok constructor with null or undefined to get that result (the type system won't allow you to construct it that way). Instead, for convenience, you can simply call ok, which will construct the type correctly.
